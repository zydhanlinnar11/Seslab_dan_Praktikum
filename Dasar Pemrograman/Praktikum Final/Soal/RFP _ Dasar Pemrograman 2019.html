Rani FINARRU Project

Time Limit2 seconds
Memory Limit64 MB

Liburan hampir tiba, sudah hampir 1 semester Rani berkuliah di Informatika. Namun sebelum liburan rani harus mengahadapi boss besar ditiap akhir semester yaitu FINALL PROJECT. Kali ini final project Rani adalah membuat sebuah database kartu untuk permainan Yu-Gi-Oh dengan memanfaatkan struct yang sudah diajarkan. Dalam permainan Yugioh ada beberapa tipe kartu yang memiliki atribute yang berbeda.

Penjelasan Tabel:
Kartu Normal memiliki data Nama,Star,Defense, Attack,Atribute,Type,Description
Kartu Fusion memiliki data Nama,Star,Defense, Attack,Atribute,Type,Description
Kartu Synchro memiliki data Nama,Star,Defense, Attack,Atribute,Type,Description
Kartu XYZ memiliki data Nama,Rank,Overlay Unit,Defense, Attack,Atribute,Type,Description
Kartu Pendullum memiliki data Nama,Scale,Star,Defense, Attack,Atribute,Type,Description
Kartu Link memiliki data Nama,Link,Linkmarker,Defense, Attack,Atribute,Type,Description
Kartu Trap dan Spell hanya memiliki Name dan Description.
Untuk setiap data yang tidak dimiliki oleh suatu type kartu tidak bisa dianggap Nol. Misalkan XYZ tidak memiliki Star, bukan berarti Star nya bernilai nol. Sorting secara default adalah nama.
 
Terdapat beberapa query dalam database Rani
 
1. Where diikuti oleh atribut Q2 kemudian value dari atribut tersebut berguna untuk memfilter kartu sesuai atribut dan nilainya.
2. SortDB diikuti Q3. Berguna untuk mengubah urutan database kartu berdasarkan atribut Q3.
3. FindOut diikuti int K dikuti Q5. Berguna mencari atribut Q5 pada urutan ke-K dalam database yang sudah diurutkan
4. Min dan Max diikuti Q4 untuk mencari data maks dan minimal dari atribut Q4.
5. Likely dikuti oleh string S1. Mencari kartu yang mengandung string S1 Description nya.
6. END Proses Query Selesai.
 

Input Format

Pada baris pertama sebuah integer N menandakan jumlah kartu yang ingin di input. Input berikutnya berupa nama kartu dan tipe kartu diikuti atribut-atribut yang sesuai oleh tipe kartu masing masing input atribut sesui urutan tabel.Kemudian dilanjutkan querry

Output Format

Untuk query Where tampilkan semua kartu yang sesuai filternya berupa nama kartu dan data dari Q3 Data tersebut. Jika tidak ada kartu yang memenuhi kriteria filter keluarkan "Tidak Ada Data Yang Cocok", Untuk Query SortDB tidak perlu output. Untuk query FindOut keluarkan Data yang dicari jika kartu tersebut tidak memiliki atribut yang dicari keluarkan "Kartu Tidak Memiliki Data Tersebut". Untuk query Min atau Max keluarkan nama kartu dan nilai dari data Q4 Jika terdapat nilai yang sama ambil data terbawah jika query min dan data teratas jika query max Untuk query Likely keluarkan semua data yang memenuhi berupa nama kartu dan deskirpsi kartu pada baris dibawahnya.

Explanation

Query Where Linkmarker Top Kartu yang memiliki Linkmarker top adalah Firewall Dragon maka outputkan nama kartu beserta datanya. Untuk data
Linkmarker tampilkan semua Linkmarker yang dimiliki kartu tersebut sesuai urutan(ada di konstrain).
Query Where Attack 2500 menampilkan semua kartu yang memiliki attack 2500 tampilkan nama dan nilai attacknya sesuai
urutan database.
Query FindOut 1 Name cari nama pada data index ke-1(base 0) kemudian tampilkan nama kartunya.
Query SortDB Attack sorting kartu berdasarkan data serangan secara descending tidak perlu melakukan output.
Query Where Attack 2500 sama seperti sebelumnya.
Query FindOut 1 Name cari nama pada data index ke-1 kemudain tampilkan nama kartunya
Query Min Attack Kartu serangan minimal adalah 2500 namun ada 4 kartu yang memiliki nilai sama maka
tampilkan yang terbawah.
Query Max Defense tampilkan kartu dengan nilai defense tertinggi.
Query Likely Blue cari kartu yang mengandung kata biru pada deskripsinya.
